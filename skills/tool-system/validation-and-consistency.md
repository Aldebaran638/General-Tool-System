# 验证与一致性契约

## 1. 前端必须验证的内容

1. route 是否可访问。
2. navigation 是否已注册。
3. 页面是否覆盖加载、空状态、成功、失败、权限不足等已定义状态。
4. API 请求路径是否与设计文档一致。
5. 前端测试是否落在 frontend/tests/<group>/<tool-key>/。
6. 构建是否通过。

## 2. 后端必须验证的内容

1. router 是否已接入。
2. service 与 repository 职责是否分离。
3. API 路径、方法、请求体、响应体是否与设计文档一致。
4. 数据模型与迁移决策是否一致。
5. 后端测试是否落在 backend/tests/<group>/<tool-key>/。
6. 目标测试是否通过。

## 3. 测试 AI 必须验证的内容

1. 前端是否越界修改。
2. 后端是否越界修改。
3. 文档与代码是否一致。
4. 前后端契约是否一致。
5. 自动化测试是否真实执行。
6. 关键链路是否至少人工或自动跑通一次。

## 4. 报告模板

前端报告至少包含：

1. 输入物路径
2. 修改文件
3. 新增文件
4. 新增或修改路由
5. 新增或修改导航
6. 调用的 API
7. 执行的验证
8. 未完成项

后端报告至少包含：

1. 输入物路径
2. 目标模块路径
3. 修改文件
4. 新增文件
5. 新增或修改接口
6. 新增或修改模型
7. 执行的验证
8. 未完成项

测试报告至少包含：

1. 已执行项
2. 失败项
3. 越界项
4. 未执行项
5. 最终是否通过

## 5. 一致性红线

以下任一项出现，任务不得视为完成：

1. 新工具未按 group/tool-key 收敛。
2. 前端伪造设计文档中不存在的 API。
3. 后端实现了任务包之外的额外业务规则。
4. 测试目录仍然散落在根目录旧位置。
5. 自动化测试未执行却声称通过。
