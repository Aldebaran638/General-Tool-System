# 工具体系共享总览

本目录定义当前项目的统一 tool system skill 基线。

目标不是提供一个抽象口号，而是为架构师、前端 AI、后端 AI 提供同一套可执行真相源，确保在上下文有限、子智能体能力有限时，仍然能稳定产出结构一致的代码。

## 适用范围

- 新增工具组
- 新增工具
- 修改现有工具
- 将历史零散页面或接口收敛为工具模块

## 核心目标

1. 统一术语。
2. 统一目录结构。
3. 统一任务包格式。
4. 统一前后端交付边界。
5. 统一验证与报告格式。

## 角色链路

1. 架构师只负责与用户对话、澄清需求、切分工具边界、生成设计与任务包。
2. 前端 AI 只负责一个工具的前端实现与前端测试。
3. 后端 AI 只负责一个工具的后端实现与后端测试。
4. 前端 AI 负责前端实现、前端测试、前端越界自检与 frontend report。
5. 后端 AI 负责后端实现、后端测试、API 契约测试、迁移校验、后端越界自检与 backend report。
6. 架构师根据前后端报告与代码 diff 做最终审查。

## 统一真相源优先级

1. 用户当前已确认的需求
2. 架构师输出的任务包
3. 架构师输出的任务清单
4. 架构师输出的设计文档
5. 架构师输出的测试文档
6. 本目录中的共享规则
7. 仓库内的本地架构说明
8. 现有代码，仅作为兼容性上下文

## 和现有代码的关系

1. 本目录定义新工作必须收敛到的结构。
2. 历史遗留的单层工具目录允许继续存在。
3. 新增工具禁止继续扩散旧结构。
4. 修改旧工具时，优先在原位置完成最小修改；只有在架构师明确要求迁移时，才执行结构迁移。
