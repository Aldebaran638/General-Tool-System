# 架构师任务包契约

架构师对前端、后端、测试 AI 的派单必须基于同一份交付包。

## 1. 交付包必须包含的字段

1. 需求摘要
2. group
3. tool_key
4. tool_name
5. 用户可见入口名称
6. 前端路由路径
7. 前端工具目录路径
8. 后端模块目录路径
9. 前端测试目录路径
10. 后端测试目录路径
11. API 契约清单
12. 权限规则
13. 数据模型决策
14. 页面状态要求
15. 成功流程与失败流程
16. 文件允许修改范围
17. 文件禁止修改范围
18. 风险与待确认项

## 2. API 契约最小字段

每个接口必须至少写明：

1. endpoint_id
2. path
3. method
4. auth_requirement
5. request_shape
6. response_shape
7. error_shape
8. business_rule_notes

## 3. 任务清单格式

任务清单必须分为四段：

1. FE 任务
2. BE 任务
3. INT 任务
4. QA 任务

每条任务必须至少写明：

1. task_id
2. title
3. scope
4. depends_on
5. done_when
6. allowed_files

## 4. 派单规则

1. 架构师向前端 AI 派单时，必须同时给出设计文档、测试文档、任务清单、允许范围。
2. 架构师向后端 AI 派单时，必须同时给出设计文档、测试文档、任务清单、允许范围。
3. 架构师向测试 AI 派单时，必须同时给出设计文档、测试文档、任务清单、前后端报告。
4. 架构师必须明确写出本次任务是否允许迁移旧工具目录。

## 5. 交付包写法要求

1. 必须面向执行，不写空泛口号。
2. 必须写出真实路径。
3. 必须写出文件边界。
4. 必须写出未决问题。
5. 必须让子智能体在没有额外杂乱上下文时也能执行。
