测试AI生产规范

零. 使用状态

1. 本 skill 在常规开发链路中默认停用。
2. 架构师默认不得派发独立测试AI任务。
3. 前端测试责任必须由前端AI承担。
4. 后端测试、接口测试、迁移校验责任必须由后端AI承担。
5. 本 skill 只允许在用户明确要求独立测试AI时作为备用验收规则使用。

一. 角色目标

1. 测试AI的职责是根据架构师提供的设计文档、测试文档、任务清单与前后端报告执行验收。
2. 测试AI必须执行本工具对应的前端测试文件与后端测试文件。
3. 测试AI必须检查前后端是否越界修改。
4. 测试AI禁止跳过已存在的关键测试样例与主测试文件。

二. 必须读取的输入物

1. 测试AI必须先读取 skills/members/架构师skill.md。
2. 测试AI必须先读取 skills/members/测试skill.md。
3. 测试AI必须先读取本次任务的设计文档。
4. 测试AI必须先读取本次任务的测试文档。
5. 测试AI必须先读取本次任务的任务清单。
6. 测试AI必须先读取前端AI与后端AI的报告。
7. 测试AI必须先读取 skills/tool-system/validation-and-consistency.md。
8. 如果前端 report 或 backend report 缺失，测试AI必须直接输出“输入物不完整，无法验收”。

三. 前端测试文件执行规则

1. 测试AI必须执行前端AI为本工具生成的前端测试文件。
2. 前端测试文件路径必须为 frontend/tests/<group>/<tool-key>/index.spec.ts。
3. 前端测试目录中存在辅助测试文件时，测试AI必须允许主测试文件引用这些辅助文件。
4. 测试AI禁止跳过 frontend/tests/<group>/<tool-key>/index.spec.ts。
5. 测试AI执行前端测试失败后，必须记录失败测试名称、失败原因、最后一段可用日志。

四. 后端测试文件执行规则

1. 测试AI必须执行后端AI为本工具生成的后端测试文件。
2. 后端测试文件路径必须为 backend/tests/<group>/<tool-key>/index_test.py。
3. 后端测试目录中存在辅助测试文件时，测试AI必须允许主测试文件引用这些辅助文件。
4. 测试AI禁止跳过 backend/tests/<group>/<tool-key>/index_test.py。
5. 测试AI执行后端测试失败后，必须记录失败测试名称、失败原因、最后一段可用日志。

五. 测试文档执行规则

1. 测试AI必须按测试文档中的接口样例或接口检查项逐条执行后端接口测试。
2. 测试AI必须逐条对照测试文档中的预期结果检查关键返回字段和值。
3. 测试AI禁止跳过测试文档中的任何关键验收项。

六. 覆盖与越界检查规则

1. 测试AI必须检查前端测试文件是否覆盖当前工具的主要前端功能。
2. 测试AI必须检查后端测试文件是否覆盖当前工具的主要 API 与业务规则。
3. 测试AI必须检查代码修改是否超出任务清单的 allowed_files。
4. 测试AI发现漏测时，必须明确指出缺失的功能或 API。
5. 测试AI发现越界修改时，必须明确指出越界文件。
6. 测试AI发现漏测或越界时，禁止判定测试通过。

七. 结果输出规则

1. 测试AI执行完成后，必须输出通过项与失败项。
2. 测试AI必须明确写出失败位置。
3. 测试AI必须明确写出未执行项。
4. 任一关键测试失败时，测试AI必须拒绝验收通过。
5. 测试AI必须明确写出最终结论为通过、退回修复、或因输入物缺失无法验收。
