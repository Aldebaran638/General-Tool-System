# Round 008 Requirements - reimbursement_excel

## 背景

系统需要根据固定模板生成报销 Excel。模板文件：

```text
docs/報銷表模板最新.xlsx
```

示例签字 PDF：

```text
docs/202512_signed.pdf
```

本轮只生成 `.xlsx`，不生成 PDF，不处理签字。

## 目标

新增 `finance / reimbursement_exports` 工具：

- 管理员筛选购买记录。
- 管理员手动勾选购买记录。
- 管理员填写导出表头配置。
- 系统按固定模板生成 Excel。
- 系统保存导出历史。
- 系统保存物理 Excel 文件。
- 物理文件默认保留 1 天，管理员可前端配置。
- 过期物理文件可清理，历史记录永久保留。

## 非目标

- 不生成 PDF。
- 不处理签字。
- 不做报销审批流。
- 不改变购买记录状态。
- 不阻止重复导出。
- 不支持多币种同一份 Excel。
- 不做汇率换算。

## 权限

### 管理员

- 可查看所有用户的可导出购买记录。
- 可筛选和勾选所有用户的购买记录。
- 可生成 Excel。
- 可查看导出历史。
- 可下载未过期的导出文件。
- 可修改导出文件保留天数。
- 可执行过期文件清理。

### 普通用户

- 无权生成报销 Excel。
- 无权查看导出设置。
- 无权修改导出设置。
- 无权清理导出文件。

## 数据来源

默认只允许导出满足以下条件的购买记录：

- `purchase_record.status = approved`
- `purchase_record.deleted_at IS NULL`
- 存在 `invoice_match.status = confirmed`
- 关联发票未逻辑删除
- 关联发票状态不是 `voided`

## 筛选条件

前端必须支持：

- 购买日期范围。
- 报销人 / 用户。
- 大类。
- 小类。
- 币种。
- 是否已导出过。
- 关键词：订单名称 / 备注。

默认筛选：

- 已批准。
- 已确认匹配。
- 未删除。
- 发票未作废。

## 手动勾选

- 管理员必须手动勾选购买记录。
- 未勾选不能生成。
- 选中记录必须全部满足导出条件。
- 选中记录必须是同一币种。
- 允许一次导出多个用户的购买记录。
- 允许重复导出已导出过的购买记录，但前端必须弹出警告。

## 导出表头配置

导出前管理员可填写：

- 报销部门。
- 事业部。
- 报销人。
- 日期。

全部非必填。为空则 Excel 对应位置留空。

另有字段：

- 使用公司/部门/项目/合同：统一填报销部门；如果报销部门为空则留空。

## Excel 模板规则

模板包含两个 sheet：

- `報銷單分类`：分类汇总页。
- `報銷單`：明细页。

必须保留：

- 合并单元格。
- 样式。
- 行高列宽。
- 打印区域。
- 公式或等价计算结果。

## 类别顺序

导出时按固定类别顺序排序：

1. 交通费用。
2. 膳食 / 应酬费用。
3. 汽车费用。
4. 其他项目费用。

组内排序：

1. `purchase_date ASC`
2. `created_at ASC`
3. `id ASC`

`ASC` 表示升序，即从早到晚、从小到大。

## 单据编号

- 每次导出重新编号。
- 从 1 开始。
- 按排序后的明细购买记录逐条递增。
- 单据编号是导出快照，不是购买记录永久字段。

示例：

```text
交通费用 22 条 -> 编号 1-22
汽车费用 1 条 -> 编号 23
其他项目费用 10 条 -> 编号 24-33
```

如果新增一条汽车费用后重新导出，它按类别顺序插入汽车费用分组，后续其他项目费用编号顺延。

## 明细页规则

`報銷單` 明细页：

- 每条购买记录一行。
- 日期：`purchase_date`。
- 金额：`amount`。
- 单据编号：导出时生成的编号。
- 用途 / 说明：优先 `remark`，没有则 `order_name`。
- 使用公司/部门/项目/合同：导出表头中的报销部门。
- 交通费用的地点由/至：留空。

大类字段映射：

| 大类 | 明细页分区 | 说明 |
|------|------------|------|
| transportation | 交通费用 | 交通工具列可填 `order_name` |
| meals_entertainment | 膳食 / 应酬费用 | 成员列可填空或 `order_name`，用途说明按规则 |
| vehicle | 汽车费用 | 目的地留空，用途说明按规则 |
| other_project | 其他项目费用 | 项目列显示小类名称 |

## 分类页规则

`報銷單分类` 汇总页：

- 交通费用：按可读说明聚合或直接按明细合计。
- 膳食 / 应酬费用：按明细合计。
- 汽车费用：按明细合计。
- 其他项目费用：按小类汇总。

其他项目费用固定小类：

- 自动导航承载车。
- 智能喷漆机器人。
- 钢筋折弯与结扎机器人。
- 生产线车队调度。
- 研发部开销。

注意：

- 小类不是一条单据。
- 小类下面的每条购买记录才是一条单据。
- 分类页小类行显示该小类下所有购买记录的金额合计和单据编号范围。

## 导出历史

导出时创建：

- `reimbursement_export`
- `reimbursement_export_item`

导出历史永久保留。

购买记录不新增“已导出”状态，也不修改原有状态。

筛选列表和历史记录中要能显示：

- 是否已导出过。
- 最近导出时间。
- 最近导出人。

## 文件存储

物理 Excel 文件保存到：

```text
runtime_data/exports/finance/reimbursement/
```

数据库只保存：

- 相对路径。
- 原始文件名。
- MIME。
- 大小。

不得把 Excel 二进制存入数据库。

## 文件保留与清理

新增全局配置表：

```text
app_setting
```

第一版配置项：

```text
reimbursement_export_retention_days = 1
```

规则：

- 管理员可在前端配置保留天数。
- 范围：1 到 365。
- 默认：1。
- 清理只删除过期物理文件。
- 不删除导出历史。
- 文件过期或已被清理后不能下载，显示“文件已过期，可重新生成”。

## API 前缀

建议：

```text
/api/v1/finance/reimbursement-exports
```

## 验收标准

- 管理员能筛选、勾选并生成 Excel。
- 普通用户不能访问生成接口。
- 多币种选中时生成失败并返回明确错误。
- 已导出记录再次导出允许，但前端有警告。
- Excel 格式接近模板，两个 sheet 都存在。
- 明细页每条购买记录一行。
- 分类页其他项目费用按小类汇总。
- 单据编号按类别顺序和组内排序生成。
- 导出历史记录保留。
- 物理文件保留天数可由管理员前端配置。
- 清理过期文件后历史仍可见，但下载不可用。
