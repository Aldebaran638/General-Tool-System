# Round 003 Requirements - frontend_i18n

本轮实现全局前端语言切换基础设施。

## 目标

系统支持三种界面语言：

- `zh-CN`：简体中文
- `en-US`：英文
- `zh-TW`：繁体中文

本轮只处理前端 UI 文案和业务字典显示，不翻译用户填写的业务数据。

## 范围

必须覆盖：

- 全局导航
- 用户菜单/个人设置入口
- 用户管理页面
- 财务工具组名称
- `purchase_records` 购买记录工具
- `invoice_files` 发票文件工具
- 常用状态、操作按钮、空状态、错误提示
- 业务字典：
  - 购买记录大类
  - 购买记录小类
  - 发票类型
  - 币种显示文案

## 非目标

- 不修改数据库。
- 不新增后端 API。
- 不把用户业务数据翻译成多语言。
- 不要求用户语言偏好跨设备同步。
- 不翻译后端错误详情，前端可先用通用错误文案兜底。

## 语言偏好

第一版使用浏览器本地保存：

- localStorage key：`app_locale`
- 默认语言：`zh-CN`
- 如果浏览器语言以 `zh-TW`、`zh-HK`、`zh-MO` 开头，默认 `zh-TW`
- 如果浏览器语言以 `en` 开头，默认 `en-US`
- 其他情况默认 `zh-CN`

## 用户入口

必须提供一个用户可见的语言切换入口。

推荐位置：

- 个人设置页面增加语言选择。

可选增强：

- 侧边栏用户菜单增加语言切换入口。

## 实现要求

- 使用稳定翻译 key，不在业务组件里散落硬编码多语言判断。
- 字典值数据库/API 仍存稳定 code。
- 组件显示时通过翻译函数把 code 转为当前语言文案。
- 新增工具后应能复用同一套 i18n 能力。
- 语言切换后当前页面应立即更新，无需刷新。

## 验收

- 用户可以在 UI 中切换 `zh-CN`、`en-US`、`zh-TW`。
- 切换后导航、核心页面标题、按钮、状态、财务工具字典文案同步变化。
- 刷新页面后保持上次选择。
- 构建通过。
- Playwright 至少覆盖语言切换和刷新保持。
